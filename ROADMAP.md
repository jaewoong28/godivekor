# GoDiveKor 포스트-MVP 로드맵

## Context
MVP(히어로, 교육과정 토글, 캘린더+일정 리스트)가 완성된 상태에서, 업계 최고 퀄리티의 홈페이지로 발전시키기 위한 상세 로드맵입니다. 단순 "다이빙 샵 홈페이지"를 넘어 **한국 다이빙 산업의 디지털 허브**를 목표로 합니다.

---

## 고객 여정 기반 설계 철학

스쿠버 다이빙 고객의 구매 여정은 독특합니다:

```
발견 → 관심 → "안전한가?" → 신뢰 → 예약 → 경험 → 재구매/추천
        ↑                    ↑        ↑              ↑
    Phase 4 콘텐츠      Phase 2 신뢰  Phase 3 결제  Phase 5 커뮤니티
```

다이빙은 **생명과 직결**되므로, 일반 쇼핑몰과 달리 "신뢰 구축" 단계가 핵심입니다.
또한 한 번 입문하면 평생 하는 취미이므로, "커뮤니티 충성도"가 LTV를 결정합니다.

## Phase 1: 전환율 극대화 — "방문자를 고객으로" (2~3주)

**목표:** 현재 사이트에는 문의/예약 경로가 아예 없음. CTA → 실제 행동으로 연결되는 파이프라인 구축.

### 1-1. 카카오톡 상담 플로팅 버튼
- 화면 우하단 상시 노출
- **이유:** 한국 사용자 95%가 카카오톡 사용. 다이빙은 안전 불안감이 높아 "전화"보다 "채팅으로 가볍게 물어보기"가 전환 장벽을 극적으로 낮춤. 체험 다이빙 고객 대부분은 20~30대로 메신저 선호.
- 기술: 카카오톡 채널 JavaScript SDK

### 1-2. 문의 폼
- Footer `#contact` 영역을 실제 폼으로: 이름, 연락처, 관심 과정(드롭다운), 메시지
- **이유:** 심야/카톡 불편한 고객도 문의 가능. 폼 데이터는 마케팅 분석 기초 자료.
- 기술: React Hook Form + Zod, `app/api/contact/route.ts` (Resend 이메일 발송)

### 1-3. 코스별 "문의하기" CTA 버튼
- 아코디언 펼침 영역에 버튼 추가, 클릭 시 폼으로 스크롤 + 해당 과정 자동 선택
- **이유:** 과정 설명을 읽고 관심이 생긴 **그 순간**에 행동으로 연결. 관심→행동 거리 = 스크롤 한 번.

### 1-4. 이미지 최적화
- `<img>` → `next/image` 전환, 실제 다이빙 사진 사용
- **이유:** 다이빙은 시각적 경험이 구매 결정의 핵심. 자체 사진은 신뢰↑, WebP 변환/lazy loading으로 LCP 50%+ 개선.
- 파일: `components/Hero.tsx`, `components/CourseSection.tsx`

### 1-5. 기본 SEO 강화
- Open Graph / Twitter Card 메타태그
- `app/robots.ts`, `app/sitemap.ts` 생성
- JSON-LD 구조화 데이터 (LocalBusiness, Course 스키마)
- **이유:** "스쿠버 다이빙 교육 서울", "PADI 오픈워터 한국" 검색 상위 노출이 신규 유입의 핵심 채널.

### 비즈니스 KPI
- 문의 전환율: 0% → 3~5%
- 모바일 즉시 상담 가능
- SEO 기반 마련 (3~6개월 후 효과 발현)

## Phase 2: 신뢰 구축 & 콘텐츠 확장 — "이 곳이면 안전하다" (2~3주)

**목표:** 다이빙 최대 구매 장벽 = 안전 두려움. "이 업체에 내 생명을 맡겨도 되는가"에 확신을 주는 신뢰 요소 전면 배치.

### 2-1. About / 강사 소개 페이지
- 강사 사진, PADI 자격번호, 경력, 다이빙 로그 수
- 업체 스토리, 교육 철학, 안전 관리 체계, PADI 공인 센터 배지
- **이유:** "누가 가르치는가"는 가격보다 중요한 의사결정 요소. PADI 번호 공개 → 고객이 직접 검증 가능 → 신뢰 극대화.
- 파일: `app/about/page.tsx`

### 2-2. 수강 후기 섹션 (Social Proof)
- 홈페이지 캐러셀: 이름(이니셜), 수강 과정, 후기, 별점
- 초기 하드코딩 → Phase 5에서 DB 연동
- **이유:** "나와 비슷한 사람이 여기서 성공적으로 자격증을 땄다"는 사회적 증거가 두려움 극복의 가장 강력한 도구. 체험 다이빙(15만원)은 후기 하나로 즉시 구매 유발.

### 2-3. 투어 상세 페이지
- 목적지별 카드: 이미지 갤러리, 일정표, 포함/불포함, 가격, 난이도
- **이유:** 투어는 교육보다 객단가 높음(100~300만원). 상세 정보 없으면 "다른 곳에서 알아볼게요". 아름다운 수중 사진이 가득한 투어 페이지 = 그 자체로 판매 도구.
- 파일: `app/tours/page.tsx`

### 2-4. 갤러리 섹션
- 교육 현장, 수중 사진, 자격증 취득 기념사진
- Masonry/Lightbox 레이아웃 + Instagram 피드 임베딩
- **이유:** 다이빙은 "경험재". 사진으로 미리 보여주는 것이 텍스트 설명보다 100배 효과적.

### 2-5. FAQ 섹션
- "수영 못해도 가능?", "자격증 기간?", "체험 다이빙 안전?" 등
- 기존 Accordion 컴포넌트 재활용
- **이유:** 입문자 80%가 같은 질문. 문의 전 자가 해결 + SEO 롱테일 키워드 보물창고.

### 비즈니스 KPI
- 체류 시간 2배 증가
- 투어 문의 증가
- 반복 질문 감소 → 운영 효율↑

## Phase 3: 예약 시스템 & 결제 — "지금 바로 신청" (3~4주)

**목표:** "문의 → 카톡 상담 → 입금 확인"이라는 수동 프로세스를 온라인 예약/결제 자동화로 전환. 매출에 가장 직접적인 기술 투자.

### 3-1. 데이터베이스 도입 (Supabase)
- 테이블: users, courses, tours, events, bookings, reviews
- 하드코딩 데이터 → DB 마이그레이션
- **이유:** 예약/결제/관리의 전제 조건. Supabase는 인증+스토리지+실시간 제공, 소규모 비즈니스에 최적.
- 기술: Supabase + Prisma ORM

### 3-2. 온라인 예약 시스템
- 캘린더 이벤트 클릭 → 예약 모달: 이름, 연락처, 인원, 자격 수준
- **잔여석 표시** (강사 1인당 4명 정원제)
- 예약 확인 이메일 + 카카오 알림톡 자동 발송
- **이유:** "자리 있을까?" → "지금 바로 예약"은 매출의 핵심. **잔여석 표시 = FOMO(놓칠까봐 두려움) 자극**. 다이빙은 소수 정원제라 이 효과가 특히 강력.
- 파일: `components/ScheduleSection.tsx` 확장

### 3-3. 한국 결제 연동
- PortOne(구 아임포트): 카드, 카카오페이, 네이버페이, 계좌이체
- 예약금 결제(전액 또는 10만원 선금) + 잔금 현장 결제 옵션
- **이유:** 한국 고객은 카카오페이/네이버페이에 익숙. "입금 확인" 수동 절차 제거 → 예약 확정률 2배↑. 예약금 → 노쇼 방지.
- 기술: PortOne v2 SDK

### 3-4. 관리자 대시보드 (기본)
- `/admin` 경로: 이벤트/스케줄 CRUD, 예약 목록, 상태 변경(대기/확정/취소)
- **이유:** 사업자가 코드 수정 없이 직접 스케줄 관리 가능해야 지속 가능.

### 3-5. Google Analytics 4 + 전환 추적
- 주요 이벤트: 페이지 뷰, 아코디언 열기, 문의 제출, 예약 완료, 결제 완료
- 네이버 서치어드바이저 등록
- **이유:** "어떤 코스가 관심받는지", "어디서 이탈하는지" 모르면 개선 불가. 전환 퍼널 분석이 비즈니스 의사결정의 기초.

### 비즈니스 KPI
- 24시간 예약/즉시 결제 → 전환율 극대화
- 노쇼율 50%↓
- 운영 자동화 → 사업자 시간 절약
- 데이터 기반 의사결정 시작

## Phase 4: 차별화 & 몰입 경험 — "다른 곳에서는 볼 수 없는" (3~4주)

**목표:** WordPress 템플릿으로는 절대 따라올 수 없는 인터랙티브 경험. "한국 다이빙 업계에서 이런 사이트는 처음이다" 반응 이끌어내기.

### 4-1. 인터랙티브 다이브 사이트 맵
- 한국(제주 문섬, 범섬, 동해 울진, 속초) + 해외(세부, 발리, 시파단) 포인트 표시
- 클릭 시: 수심, 시야, 난이도, 추천 시즌, 대표 생물, 사진
- **이유:** "어디서 다이빙 가능한지" 탐색 = 입문자의 핵심 관심사. 콘텐츠 마케팅 자산 + 투어 판매 연결 고리. **이런 인터랙티브 맵을 가진 한국 다이빙 사이트는 현재 없음.**
- 기술: `react-leaflet` 또는 `mapbox-gl`

### 4-2. 실시간 해양 날씨/컨디션 위젯
- 주요 포인트 수온, 파도 높이, 시야, 바람 정보
- "이번 주말 제주 다이빙 가능할까?" 즉시 확인
- **이유:** 다이버에게 해양 컨디션 = 출발 여부 결정의 핵심 정보. 이것만으로 **매일 사이트 방문 이유** 제공. 다이버 커뮤니티에서 브랜드 인지도 급상승.
- 기술: 기상청 해양 API + Open-Meteo Marine API (무료)

### 4-3. 360도 수중 갤러리 / VR 미리보기
- GoPro MAX 촬영 360도 수중 영상/사진, 웹 VR 뷰어
- **이유:** 다이빙 미경험자에게 "바다 속이 이렇게 아름답다" 보여주기 = 가장 강력한 동기 부여. 유튜브 360보다 사이트 내 직접 경험이 전환에 더 효과적.
- 기술: `pannellum` (경량 360 뷰어)

### 4-4. 코스 진행 타임라인 시뮬레이터
- "오픈워터 코스 신청하면 어떤 과정?" → 시각적 타임라인
- Day 1: 이론 → Day 2~3: 수영장 → Day 4: 바다 → 자격증
- 각 단계별 사진/영상 포함
- **이유:** 미지의 과정에 대한 불안감 해소. "내가 뭘 하게 되는지 미리 알면 안심" → FAQ보다 훨씬 직관적이고 강력.
- 기술: Framer Motion 애니메이션

### 4-5. 블로그/콘텐츠 허브
- MDX 기반: 다이빙 팁, 장비 리뷰, 투어 후기, 해양 생물 도감
- **이유:** SEO의 근본적 해결책. "스쿠버 다이빙 비용", "PADI 오픈워터 후기" 등 검색어 유입 콘텐츠가 지속적으로 쌓여야 함.
- 기술: `next-mdx-remote`, 파일: `app/blog/`

### 비즈니스 KPI
- "한국 다이빙 업계에서 이런 사이트는 처음" 브랜드 차별화
- 재방문율 3배 (해양 날씨 위젯)
- SEO 장기 성장 (블로그)
- SNS 바이럴 (360도 콘텐츠)

## Phase 5: 커뮤니티 & 충성도 — "다이버들의 놀이터" (4~5주)

**목표:** 일회성 고객 → 평생 고객 + 커뮤니티 멤버. 반복 구매(투어) + 자발적 추천(입소문) 엔진 구축.

### 5-1. 회원 시스템 & 마이페이지
- 카카오/네이버 소셜 로그인
- 마이페이지: 예약 내역, 보유 자격증, 다이빙 이력
- **이유:** 소셜 로그인 선호. 마이페이지 = "내 다이빙 여정" + 다음 과정/투어 추천 기반.

### 5-2. 디지털 다이빙 로그북
- 각 다이빙 기록: 날짜, 장소, 수심, 수온, 메모, 사진
- GoDiveKor 참여 시 자동 로그 생성 + 누적 시각화
- **이유:** 종이 로그북 대체 → **매일 방문 이유**. 로그 데이터 → "다음 과정 추천" 근거 (50회+ → 다이브마스터 추천).

### 5-3. 게이미피케이션 (뱃지 & 레벨)
- 자격증별 뱃지 (OW, AOW, Rescue, DM)
- 다이빙 횟수 뱃지 (10회, 50회, 100회)
- 목적지 뱃지 ("제주 정복", "세부 마스터")
- **이유:** "다음 뱃지를 위해 다음 투어 가야지" 동기 부여. SNS 뱃지 공유 → 비용 없는 바이럴. 다이빙 커뮤니티는 경험 자랑/공유 문화가 강함.

### 5-4. 버디 파인더 (동행 찾기)
- 투어 참가 확정 다이버 목록(동의 기반) + "함께 갈 사람 찾기"
- **이유:** "같이 갈 사람 없어서" 포기하는 경우 많음. 버디 매칭 → 개인 참가율↑ + 커뮤니티 소속감 강화.

### 5-5. 실제 후기 시스템 (UGC)
- 코스/투어 완료 후 자동 후기 요청 (이메일 + 알림톡)
- 사진 첨부 + 별점 + 구글 리뷰 연동 유도
- **이유:** Phase 2 하드코딩 후기 → 진짜 UGC로 대체. 진짜 후기는 가짜보다 10배 강력. 구글 리뷰 = 로컬 SEO 직접 영향.

### 5-6. 다국어 지원 (한/영/일)
- **이유:** 제주/동해 방문 외국인 다이버 증가 추세. 영어 사이트 → 해외 커뮤니티 인지도.
- 기술: `next-intl`

### 비즈니스 KPI
- 고객 LTV 극대화: OW → AOW → 투어 반복 여정을 시스템이 유도
- 자연 바이럴: 뱃지 공유, 후기, 버디 초대
- 재구매율 40%+
- 플랫폼화: 홈페이지 → 다이버 커뮤니티

## Phase 6+: 플랫폼 확장 (지속적)

- **온라인 이론 교육 (LMS)**: 동영상 강의 + 퀴즈 + 블렌디드 러닝
- **장비 쇼핑몰**: 초기 큐레이션 → 자체 판매
- **PWA 모바일 앱**: 오프라인 지원, 푸시 알림
- **AI 챗봇**: FAQ 기반 상담 + 코스/투어 추천

## 검증 방법
- 각 Phase 완료 시 `npm run build` + `npm run dev`로 전체 동작 확인
- Phase 1: 카카오톡 버튼 클릭 → 채팅 연결, 문의 폼 제출 → 이메일 수신 확인
- Phase 2: About/Tours 페이지 네비게이션, 후기 캐러셀 동작
- Phase 3: 예약 → 결제 → 확인 메일 플로우 E2E 테스트
- Phase 4: 지도 인터랙션, 날씨 API 응답, 360 뷰어 로드
- Phase 5: 소셜 로그인 → 마이페이지 → 로그 작성 → 뱃지 획득 플로우
